# 文献透镜 / PaperLens v0.4.0 发布说明

## 版本定位

v0.4.0 是产品化优化版。该版本不继续堆叠新功能，而是围绕可交付、可试用和发布前收口进行打磨，为 v0.5 的完整验收和打包发布做准备。

## 优化内容

- UI / 交互：优化多论文对比页说明、综述草稿区域、导出按钮 loading 和空状态。
- 错误提示：补充 PDF、导出、综述保存、API 调用等场景的中文可读提示。
- 性能优化：Word 导出改为动态导入 `docx`，减少首屏 bundle 压力。
- 全局搜索：保持空关键词不触发搜索、搜索结果数量限制和 debounce。
- 数据安全：补充 `.gitignore`，覆盖 `.env`、本地数据库、SQLite 文件、临时文件和 release/target 产物。
- API 配置：继续强调非 custom_api 模式不会调用模型，Provider 需要用户真实 Key 验证。
- 导出体验：取消导出不作为错误，导出成功显示路径，AI 内容保留核对提醒。
- 文档：README、使用说明、发布检查清单和作品集介绍补充 v0.4 状态。

## 安全说明

- 不写死 API Key。
- 不在 Markdown / Word 导出中写入 API Key、Base URL 或 Provider 配置。
- 诊断日志只保存脱敏摘要。
- AI 输出和综述草稿不保存完整请求体或 API Key。
- 用户论文默认保存在本地 SQLite 和本地文件系统。

## 已知限制

- OCR 仍为预留能力，不是完整 OCR 实现。
- 本地模型模式仍以说明和 OpenAI-compatible 路径为主，Ollama adapter 未正式实现。
- Word 导出是基础格式，不是复杂论文模板。
- 多论文对比和综述草稿质量依赖论文解析质量、用户笔记和模型能力。
- Provider 可用性仍需用户使用自己的 API Key、Base URL 和 Model Name 验证。
