# 文献透镜 / PaperLens v0.2.0 发布说明

## 版本定位

v0.2.0 是能力增强版，在 v0.1 本地闭环基础上增强 Provider 诊断、AI 结果回溯、问答证据定位、原 PDF 打开和 Word 导出。

## 新增能力

- Provider 状态 / 调用诊断面板。
- 测试连接结果详情：测试时间、Provider、Model、Adapter、请求状态、错误类型和脱敏摘要。
- 本地诊断日志表 `llm_call_logs`，记录最近调用状态，不保存 API Key、完整 Prompt 或论文全文。
- AI 结果历史表 `ai_outputs`，结构化保存论文信息提取、精读笔记和论文问答结果。
- 论文问答 evidence 结构化保存，支持显示 chunk、章节标题和原文片段。
- evidence 跳转到对应 chunk，并短暂高亮。
- “打开原 PDF”按钮，使用系统默认 PDF 阅读器打开原文件。
- Word `.docx` 导出，包含论文信息、本地分析、分块、笔记、AI 结果、问答记录和证据 chunk。

## 数据库变化

新增表：

- `llm_call_logs`
- `ai_outputs`

升级方式：

- 使用 `CREATE TABLE IF NOT EXISTS`，不会清空旧数据。
- 旧问答记录的 `paper_qa.evidence` 字段仍兼容普通字符串；新问答会保存 JSON evidence。

## Provider 诊断说明

诊断日志仅保存：

- Provider
- Adapter
- Model
- Base URL
- Action
- Status
- Error Type
- 脱敏 message
- Created At

不会保存：

- API Key
- 完整 Prompt
- 完整论文全文
- 完整模型 raw response

## 当前限制

- PDF 原文预览当前为“使用系统默认阅读器打开”，暂未做内嵌 PDF.js 预览。
- evidence 定位基于 chunk，不等于精确 PDF 页码。
- Word 导出为基础格式，不是复杂论文模板。
- Provider 仍需用户使用真实 API Key、Base URL 和 Model Name 手动验证。
- 不支持扫描版 PDF OCR。
- 不做云同步、账号系统或默认服务器。

## 推荐验收流程

1. 打开设置页，查看 Provider 状态 / 调用诊断面板。
2. 在缺少 API Key 的情况下测试连接，确认失败日志写入。
3. 使用真实配置测试连接，确认成功日志写入。
4. 清空诊断日志。
5. 进入已解析论文详情页，执行论文信息提取、精读笔记生成和论文问答。
6. 检查 AI 结果历史是否新增记录。
7. 检查问答 evidence 是否显示 chunk 和片段。
8. 点击 evidence，确认跳转并高亮对应 chunk。
9. 点击“打开原 PDF”，确认系统阅读器打开文件。
10. 导出 Markdown 和 Word，确认不包含 API Key。
