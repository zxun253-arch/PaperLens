# 文献透镜 / PaperLens 作品集介绍

## 一句话介绍

文献透镜是一款本地优先的 Windows 桌面端论文辅助工作台，帮助用户完成 PDF 导入、文本解析、本地分析、Prompt 工作流、阅读笔记管理和 Markdown 导出。

## 项目背景

本科生、研究生和科研初学者在阅读论文时，经常需要在 PDF、笔记工具和 AI 网页之间反复切换。文献透镜希望把论文阅读流程整理成一个可持续使用的本地工作台，同时保留用户对 AI 服务商和数据边界的控制权。

## 项目目标

- 跑通单篇论文的本地阅读闭环。
- 不依赖默认服务器，不强制接入 AI。
- 在无 API 情况下提供本地规则分析和 Prompt 工作流。
- 在用户自配 API 后支持论文信息提取、精读笔记和论文问答。
- 将阅读过程沉淀为可导出的 Markdown 记录。

## 技术栈

- Tauri
- Rust
- React
- TypeScript
- Vite
- Tailwind CSS
- SQLite
- pdf-extract

## 我实现的核心模块

- Tauri 桌面端项目初始化与打包配置
- React 页面路由与论文工作台 UI
- SQLite 数据库初始化和读写封装
- PDF 文件导入与本地文件信息保存
- Rust 侧 PDF 文本解析命令
- 论文分块保存与展示
- 本地分析模块：统计、章节结构、关键词、关键句、搜索
- Prompt 模板与 Prompt 构建流程
- 阅读笔记管理：手动笔记、外部 AI 回填、AI 生成笔记
- Markdown 导出模块
- 多 Provider 自定义大模型 API 架构
- Provider 调用诊断和本地日志
- AI 结果历史结构化保存
- 问答 evidence chunk 定位与高亮
- Word 文档导出

## 项目亮点

- 本地优先：论文、分块和笔记默认保存在本地。
- AI 可选：无 API 也能完成 PDF 解析、本地分析、Prompt、笔记和导出。
- Prompt 工作流：适合没有服务器和默认模型额度的 MVP 阶段。
- 多 Provider 架构：支持 OpenAI-compatible、Claude、Gemini 等服务商扩展。
- 结构化沉淀：阅读过程可以导出为 Markdown，用于 Obsidian、Notion、Word 等工具。
- 可追溯问答：AI 回答保存 evidence chunk，便于回到论文原文核对。
- 本地诊断：Provider 调用日志保存在本地，辅助用户排查 API 配置问题。

## 技术难点

- Tauri 桌面端文件权限、对话框和本地路径处理。
- PDF 文本层解析质量不稳定。
- 论文分块需要兼顾章节识别和通用回退策略。
- Prompt 长度需要控制，避免上下文过长。
- API Provider 返回格式不统一，需要统一入口和错误处理。
- 本地算法结果需要和 AI 生成内容明确区分。

## 解决方案

- 使用 Tauri 插件和 Rust command 处理本地文件与 PDF 解析。
- 使用 SQLite 保存论文、分块、笔记、问答和设置。
- 对 PDF 文本先做章节识别，失败时按长度和段落回退分块。
- Prompt 构建时优先选择摘要、引言、方法、结果、讨论、结论等核心分块。
- 统一封装 LLM Provider，页面只调用统一服务层。
- 在 UI 和 Markdown 导出中明确标注本地分析、外部 AI 回填和 AI 生成内容。

## 可展示的能力

- 导入并解析本地 PDF。
- 查看论文分块和本地分析结果。
- 生成适合外部 AI 使用的 Prompt。
- 保存手动笔记和外部 AI 回填内容。
- 使用自定义 API 完成论文信息提取、精读笔记和论文问答。
- 导出结构化 Markdown / Word 阅读记录。
- 查看 AI 结果历史和问答证据 chunk。

## 后续优化方向

- 优化扫描版 PDF 的 OCR 支持。
- 提升复杂论文排版解析效果。
- 增强本地关键词与关键句算法。
- 提供更安全的 API Key 存储方案。
- 增加本地模型模式，支持 Ollama / LM Studio。
- 支持多论文主题整理和文献综述工作流。

# v0.3 项目亮点补充

v0.3 将文献透镜从单篇论文辅助工具扩展为研究工作流辅助工具：

- 文献库管理：标签、阅读状态、重点收藏、筛选和排序。
- 全局检索：跨论文搜索元信息、标签、分块、笔记和问答。
- 多论文对比：基于元信息、本地关键词和用户笔记生成对比表与 Prompt。
- 文献综述辅助：支持综述 Prompt、自定义 API 生成草稿、本地保存和 Markdown / Word 导出。
- OCR 预留：清晰区分扫描版 PDF 处理边界，不默认上传云端。
- 本地模型预留：说明 Ollama / LM Studio 接入方向，保持本地优先架构。

这部分体现了产品从“文件解析 + AI 辅助”向“科研资料组织 + 研究写作工作流”的演进。
