# 文献透镜 / PaperLens 使用说明

## 1. 启动应用

开发环境启动命令：

```powershell
$env:Path = 'C:\Users\49958\.cargo\bin;' + $env:Path
npm.cmd run tauri dev
```

启动后进入论文库页面。

## 2. 导入 PDF

1. 在论文库页面点击“导入 PDF”。
2. 在系统文件选择窗口中选择 `.pdf` 文件。
3. 导入成功后，论文会出现在论文列表中。

当前版本只保存 PDF 路径、文件名、文件大小和导入时间，不会复制 PDF 原文件。

## 3. 解析 PDF

1. 点击论文卡片中的“查看详情”。
2. 在论文详情页点击“解析 PDF”。
3. App 会读取本地 PDF 文本层，并写入论文分块。

如果提示“未能提取到有效文本”，通常说明 PDF 可能是扫描版或文本层质量较差。当前版本暂不支持 OCR。

## 4. 查看本地分析

解析完成后，论文详情页会显示本地分析结果，包括：

- 论文基础统计
- 章节结构概览
- 本地关键词
- 本地关键句
- 论文内容搜索

这些结果由本地规则和文本算法生成，不是 AI 总结。

## 4.1 查看原 PDF 与 chunk 定位

论文详情页左侧提供“打开原 PDF”按钮，会调用系统默认 PDF 阅读器打开原文件。

注意：

- 当前版本不做复杂内嵌 PDF 预览。
- 如果 PDF 文件被移动或删除，App 会提示原路径不可用。
- 章节导航和问答证据可以跳转到对应 chunk，并在分块视图中短暂高亮。

## 5. 生成 Prompt

在论文详情页的 Prompt 工作流区域选择 Prompt 类型：

- 论文信息提取
- 中文精读笔记
- 论文问答
- 文献综述辅助

如果选择论文问答 Prompt，需要先输入问题。

## 6. 复制 Prompt 到外部 AI

生成 Prompt 后，点击复制按钮，将 Prompt 粘贴到 ChatGPT、Claude、DeepSeek、Qwen 等外部 AI 工具中使用。

Prompt 辅助模式不需要 API Key，也不需要服务器。

## 7. 回填外部 AI 结果

外部 AI 返回结果后，可以复制结果并粘贴到论文详情页的阅读笔记区域。

建议保存为“外部 AI 回填结果”，并在正式写作前结合论文原文核对。

## 8. 保存阅读笔记

阅读笔记支持：

- 手动笔记
- 外部 AI 回填结果
- AI 生成笔记

可以填写标题、选择类型、输入 Markdown 纯文本内容并保存。保存后会写入本地 SQLite。

## 9. 导出 Markdown

在论文详情页点击“导出 Markdown”。

导出内容包括：

- 论文基本信息
- 本地分析结果
- 论文分块内容
- 阅读笔记
- 导出信息

未解析论文也可以导出，但本地分析和分块区域会显示暂无内容。

## 9.1 导出 Word

论文详情页支持导出 Word `.docx` 文件。

Word 内容包括：

- 论文基本信息
- 本地分析结果
- 章节结构概览
- 本地关键词和关键句
- 论文分块内容
- 阅读笔记
- AI 结果历史
- 论文问答记录
- 问答 evidence chunk
- 导出信息

Word 导出为基础格式，适合后续继续整理，不是复杂论文模板。

## 10. 配置自定义 API

进入设置页，选择“自定义大模型 API 模式”，配置：

- Provider
- API Base URL
- API Key
- Model Name

注意：

- 不要泄露 API Key。
- 不同服务商的 Base URL 和模型名以官方控制台为准。
- 当前版本不提供默认模型额度。
- API 调用产生的费用由用户自己的服务商账户承担。
- API Key 当前保存于本地 SQLite，适合个人本地使用。

## 11. 选择 Provider

当前 Provider 状态：

- OpenAI-compatible：适合兼容 OpenAI Chat Completions 的服务。
- OpenAI、DeepSeek、Qwen、OpenRouter、Moonshot、智谱 GLM：复用 OpenAI-compatible 调用格式。
- Claude / Anthropic：使用 Anthropic 格式。
- Gemini / Google AI：使用 Gemini 格式。
- Ollama / 本地模型：后续支持，当前为占位。

部分 Provider 需要使用者结合真实 API Key、Base URL 和模型名称手动验证。

## 12. 测试连接

在自定义 API 模式下填写配置后，点击“测试连接”。

测试请求会发送最小文本请求。成功时显示连接成功，失败时会显示可读错误。页面不会显示完整 API Key。

设置页还提供“Provider 状态 / 调用诊断”区域，会显示当前模式、Provider、Adapter、API Key 是否填写、测试结果和最近 10 条诊断日志。

诊断日志不会保存 API Key、完整 Prompt、完整论文全文或完整模型原始响应。

## 13. 常见错误排查

- App 启动失败：检查 Node.js、npm、Rust、Cargo 和 Tauri 依赖是否可用。
- PDF 导入失败：确认文件存在且扩展名为 `.pdf`。
- PDF 解析失败：确认 PDF 带文本层；扫描版 PDF 当前不支持 OCR。
- 解析结果乱码：通常与 PDF 文本层编码或排版有关。
- Prompt 为空：确认论文已经解析并生成分块。
- 复制失败：检查系统剪贴板权限或重试。
- 笔记保存失败：检查数据库初始化是否正常。
- Markdown 导出失败：确认保存目录可写，文件名没有非法字符。
- API 测试失败：检查 Provider、Base URL、API Key、Model Name 和服务商额度。
- 重启后数据丢失：检查应用数据目录和 SQLite 文件是否可写。

# v0.3 研究工作流补充

## 文献库管理

- 在论文库中可以给论文添加或删除标签。
- 可以设置阅读状态：未读、阅读中、已读、归档。
- 可以将论文标记为重点收藏。
- 可以按关键词、标签、阅读状态、收藏状态筛选，并按最近导入、标题、阅读状态、解析状态排序。

## 全局搜索

论文库顶部搜索框会同时搜索：

- 论文标题、作者、摘要、关键词、文件名
- 标签
- 论文分块内容
- 阅读笔记
- 论文问答记录

点击搜索结果可以进入对应论文详情页；如果命中 chunk，会携带 chunk 定位参数并高亮对应分块。

## 多论文对比

1. 在论文库勾选 2-5 篇论文。
2. 点击“对比选中论文”。
3. 查看多论文对比表。
4. 复制“多论文对比 Prompt”到外部 AI，或在自定义 API 模式下直接生成对比结果。

## 文献综述辅助

在多论文对比页面可以：

- 生成文献综述 Prompt
- 在 custom_api 模式下生成综述草稿
- 手动粘贴外部 AI 结果
- 保存综述草稿到本地
- 导出综述 Markdown / Word

AI 相关内容仅作为写作辅助，正式使用前请结合论文原文核对。

## OCR 和扫描版 PDF

v0.3 不内置完整 OCR。遇到扫描版或图片型 PDF 时，应用会提示可能缺少文本层，并建议：

- 使用带文本层的 PDF
- 先用外部 OCR 工具处理后重新导入
- 等待后续版本的本地 OCR 或用户自定义 OCR 服务能力

应用不会默认上传 PDF 到云端 OCR。

## 本地模型模式

本地模型模式当前仍是增强预留：

- Ollama 常见本地地址：`http://localhost:11434`
- LM Studio 可通过 OpenAI-compatible 模式接入
- 本地模型效果取决于用户本机环境、模型能力和显存

## v0.4 产品化使用提示

- 非 custom_api 模式下，文献透镜不会自动调用任何模型。
- API Key 仅适合个人本机使用，请勿在公共电脑中保存。
- 导出 Markdown / Word 不会包含 API Key、Base URL 或 Provider 配置。
- Word 导出为基础格式，适合继续在 Word、WPS、Notion 或 Obsidian 中整理。
- 如果导出失败，请检查保存目录是否可写，或目标文件是否正被其他程序占用。

## v0.5 验收说明

v0.5 不新增业务功能，重点是完整验收和产品导出。建议先按发布检查清单完成本地流程测试，再使用自己的 API Key 分批验证 OpenAI-compatible、Anthropic / Claude 和 Gemini / Google AI 等 Provider。验收时不要把真实 API Key 写入截图、文档或提交记录。
