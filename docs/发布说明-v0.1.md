# 文献透镜 / PaperLens v0.1.0 发布说明

## 版本定位

v0.1.0 是文献透镜的 MVP 预览版，目标是验证本地优先的论文辅助工作流：

导入 PDF -> 解析 PDF -> 本地分析 -> 生成 Prompt -> 回填笔记 -> 导出 Markdown。

当前 v0.1.0 可提供便携版 release exe。MSI / NSIS 安装包尚未生成，原因是当前环境访问 GitHub release 受网络 / 权限限制，Tauri 无法下载 Windows 打包所需的 WiX / NSIS 工具链。这不是代码构建失败。

便携版目录：

```text
release/PaperLens-v0.1.0-portable/
```

启动方式：

```text
双击 paper-lens.exe
```

## 已知限制

- 当前仅支持带文本层的 PDF。
- 复杂排版、双栏论文的解析质量依赖 PDF 文本层。
- 本地分析基于规则和文本统计，不能替代大模型理解。
- API Key 当前保存在本机 SQLite，适合个人本地版。
- 自定义 API 调用费用由用户自己的服务商账户承担。
- Provider 的 Base URL、模型名称和计费规则以各服务商控制台为准。
- 当前提供便携版预览包；正式 MSI / NSIS 安装包需要在网络可访问 GitHub release 的环境下重新构建。

## 打包状态

已生成 release exe：

```text

```

便携版发布目录：

```text
release/PaperLens-v0.1.0-portable/
```

```text

```

失败原因：

- WiX：Tauri 尝试下载 `wix314-binaries.zip` 时被当前网络 / 权限环境阻止。
- NSIS：Tauri 尝试下载 `nsis-3.11.zip` 时同样受网络 / 超时影响。
- 该问题与业务代码无关。

后续可在网络可访问 GitHub release 的环境下重新运行：

```powershell
$env:Path = 'C:\Users\49958\.cargo\bin;' + $env:Path
npm.cmd run tauri build
```

## 安全提醒

- 不要把真实 API Key 写入代码、文档或截图。
- 不要在公共电脑中保存 API Key。
- 导出 Markdown 不包含 API Key 或 API 配置信息。
- 外部 AI 回填和 AI 生成内容仅作为阅读辅助，正式写作前请结合论文原文核对。

## 推荐测试流程

1. 启动应用。
2. 导入一篇带文本层的 PDF。
3. 解析 PDF。
4. 查看本地分析、分块内容和搜索结果。
5. 切换到 Prompt 辅助模式。
6. 生成中文精读笔记 Prompt 并复制。
7. 将外部 AI 结果回填为阅读笔记。
8. 新建一条手动笔记。
9. 导出 Markdown。
10. 重启应用，检查论文、分块、笔记和问答记录是否仍然存在。
11. 如需测试 API，使用自己的 API Key 验证 Provider 连接和论文 AI 功能。
